# /brief - Pre-Prompt Clarification

> **Philosophy**: Measure twice, cut once. Clarify before building.

## When to Use

- Complex multi-step requests
- Underspecified ideas
- Anything touching SOVEREIGN data
- Research requests that will consume significant time

## Core Brief Checklist

1. **OUTCOME**: What will be true when successful?
2. **CONTEXT**: Who is this for? What does Cilla need?
3. **CONSTRAINTS**: Must-haves, must-nots, scope boundaries
4. **DONE**: What does "finished" look like?
5. **BUDGET**: Quick (5m) / Medium (20m) / Deep (1h+)

## Rules

- If request is simple and clear: skip the brief, just execute
- If request touches health/genetics: ALWAYS brief first (SOVEREIGN data protection)
- If Cilla says "just do it": execute with Combat Protocol active (push back if needed)
- No 50-question interview dumps. Max 5 clarifying questions per turn.

## Output

After brief is complete:
```
Brief locked. Executing with:
- Outcome: [X]
- Budget: [Y]
- Constraints: [Z]
```
