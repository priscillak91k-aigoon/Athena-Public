# Protocol 116: Multi-Layer Communication (5L Density)

> **Source**: Zero-Point Codex  
> **Domain**: Communication / Social Physics  
> **Priority**: ⭐⭐ High

---

## Core Principle

> **When high-level communicators interact, every statement simultaneously carries 5 layers of information.**

Most people only track L1 (literal). Masters track all 5 in real-time.

---

## The 5 Layers

| Layer | Function | Audience | Weight | Example |
|-------|----------|----------|--------|---------|
| **L1: Literal** | Surface content | Witnesses/outsiders | 5-10% | "I learned so much today" |
| **L2: Tactical** | Immediate exchange | Direct counterpart | 15-25% | Establishes false rapport for later move |
| **L3: Frame** | Relationship structure | Both parties | 30-40% | Submission signal (lowers threat) |
| **L4: Meta** | Commentary on game | High-awareness observers | 20-30% | "We both know this is performance" |
| **L5: Rule** | Acceptable boundaries | System level | 5-10% | Implicit rules being established/violated |

---

## Classic Example: "Thanks, Bank Robber" (Inside Man)

```
Scene: Detective thanks robber for relationship advice

Single line carries all 5 layers simultaneously:

L1 (Literal 5%):
└─ Thanks for the advice

L2 (Tactical 15%):
└─ "Learning from a criminal, how absurd" (ironic)

L3 (Frame 30%):
└─ Establishes false rapport for later handshake move

L4 (Meta 30%):
└─ Submission signal—makes self look unthreatening

L5 (Rule 20%):
└─ Both know this is performance for hostages/cameras

All five layers true simultaneously ✅
```

## Human Working Memory Limitation (4-7 Chunks)

**Why multiple viewings are required to understand complex media:**

| Viewing | Cognitive Load | Focus | Understanding |
|---------|---------------|-------|---------------|
| 1st | 100% | Track literal dialogue + basic action | L1-L2 (surface) |
| 2nd | 20% | Dialogue familiar → notice body language | L2-L3 (tactical) |
| 3rd | 10% | Dialogue + action familiar → see frame battles | L3-L4 (frame) |
| 4th | 5% | Everything familiar → see entire orchestration | L4-L5 (system) |
| 5th | <5% | Auto-process → appreciate execution artistry | Extract universal principles |

**Each viewing releases cognitive bandwidth to see deeper layers ✅**

## 5-Question Meta-Communication Analysis

Run this when encountering complex interaction:

```
Q1: What's being SAID? (L1 - Content)
├─ Surface conversation
├─ Literal words
└─ Observable events

Q2: What's being DONE? (L2 - Action)
├─ Behavioral patterns
├─ Revealed preference
└─ Physical movements

Q3: What are they ACTUALLY optimizing for? (L3 - Meta)
├─ Hidden objectives
├─ Frame control attempts
└─ Power dynamics

Q4: What does this reveal about their model of each other? (L4 - Meta-Meta)
├─ What do they know the other knows?
├─ Shared implicit knowledge
└─ Mutual modeling level

Q5: What implicit rules/frames are being established or violated? (L5 - System)
├─ Interaction meta-rules
├─ Protocol boundaries
└─ Acceptable vs unacceptable moves
```

## Practical Applications

### Reading a Meeting

```
Boss says: "Let's explore other options" (L1)

Full analysis:
├─ L1: Stated interest in alternatives
├─ L2: Buying time, not deciding
├─ L3: Rejecting your proposal without direct "no"
├─ L4: Expects you to understand the soft rejection
└─ L5: Establishing "I don't say no directly" norm

Correct response: Treat as rejection, don't push.
```

### Reading a Date

```
They say: "I'll let you know about next week" (L1)

Full analysis:
├─ L1: Literal statement about scheduling
├─ L2: Creating distance
├─ L3: Soft rejection (2x soft reject = hard reject)
├─ L4: Expects you to read between lines
└─ L5: "I don't confront, you should read signals"

Correct response: Don't follow up. Move on.
```

### Reading Negotiation

```
They say: "That's an interesting proposal" (L1)

Full analysis:
├─ L1: Acknowledgment of proposal
├─ L2: Not committing
├─ L3: Maintaining leverage
├─ L4: "I've heard this before, doesn't impress me"
└─ L5: "Prove more value before I engage"

Correct response: Probe for actual objection.
```
