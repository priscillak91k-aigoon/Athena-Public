🗺️ Network Topology — How It All Fits

CONTACT NZ FIBRE (ONT) │ ▼ TP-LINK MESH ROUTER (Main) ← Acts as primary router, DHCP master 192.168.1.1 │ Subnet: 192.168.1.0/24 │ [LAN Port]──────────[Ethernet Cable] │ ▼ ASUS RT-AC86U (SJ's Room) ← Running in ROUTER mode 192.168.1.x WAN │ SJ Subnet: 192.168.2.0/24 │ ┌──────────┼──────────┐ ▼ ▼ ▼ [SJAI LAN1] [LAN2] [Wi-Fi 5GHz] (Supercomputer) (Future) (SJ's devices)
Why this topology?

SJ's ASUS is functioning as a dedicated router for the AI supercomputer lab. The ethernet cable from the TP-Link main router plugs into the WAN port of the ASUS. This gives SJ her own subnet (192.168.2.x), her own QoS controls, her own VPN, her own firewall — completely independent of the rest of the house network. The AI supercomputer gets sub-1ms intra-LAN latency. Zero wireless hops. Zero performance loss.

⚠️ Double NAT — Not a Problem Here

Double NAT (two routers doing NAT) is normally bad for gaming consoles and peer-to-peer apps. For SJ's use case — AI compute workloads, SSH, API calls, remote access — it is irrelevant. The ASUS is the only device she'll be tunnelling through, and outbound connections (which is what AI workloads do) are never affected by double NAT. Only port-forwarding inbound connections are affected, and that's solved with a simple DMZ or port forward on the TP-Link pointing to the ASUS WAN IP.

⚙️ Router Mode vs AP Mode — The Decision

✅ ROUTER MODE (Chosen)

  • Full QoS for the supercomputer
  • VPN server — remote access to SJAI
  • OpenVPN client — route all SJ traffic through VPN
  • Independent DHCP — SJ controls 192.168.2.x
  • AiProtection + full firewall active
  • Download Master (USB NAS) works
  • Traffic logging per device
  • USB app features all available

❌ AP MODE (Not Chosen)

  • Loses QoS (TP-Link handles it, poorly)
  • Loses VPN server capability
  • Loses DHCP control
  • Loses independent firewall
  • Loses Download Master
  • AiProtection disabled
  • SJ is on same subnet as house
  • Fine for extending Wi-Fi, not for lab work
Setup instruction

Plug the ethernet cable from the TP-Link into the WAN (blue/yellow) port of the ASUS. The ASUS will recognise the TP-Link as its upstream provider and request an IP via DHCP. It will then hand out 192.168.2.x addresses to everything plugged into its LAN ports. SJAI goes into LAN port 1.

Cable Quality Matters

For the WAN backhaul cable, use Cat 6 minimum. The router supports gigabit speeds — a Cat 5 cable will cap you at 100 Mbps. Cat 6 = full gigabit. If it's a long run through walls, Cat 6A is even better (less cross-talk).

⚡ QoS — Traffic Priority for the Supercomputer

Adaptive QoS Setup

Navigate: Advanced Settings → Adaptive QoS

  1. Enable Adaptive QoS. Select type: Custom (not Gaming/Streaming presets).
  2. Set priority order: Gaming → Streaming → Web Surfing → File Transfer → Others. The "Gaming" category catches low-latency UDP traffic — which is what AI model API calls often look like.
  3. In Bandwidth Limiter, set no limits for SJAI's MAC address. This tells the router to always give the supercomputer whatever it needs, never throttle it.
  4. Turn on Traffic Monitor — lets SJ see real-time bandwidth usage per device. Essential for monitoring AI workload network profiles.
⚠️ WTFast is dead — skip it

WTFast's Gamers Private Network integration reached end-of-service in 2024 and no longer works from the router interface. Don't spend time trying to configure it. If SJ wants game acceleration, the WTFast PC app still works at the client level, but it's irrelevant for AI compute workloads.

IP Traffic Controller (Advanced)

For fine-grained control: Advanced Settings → Traffic Manager → IP Traffic Controller

You can set SJAI's static IP (192.168.2.10 recommended) as Highest priority with uncapped bandwidth. All other devices fall below. This is the surgical option — SJAI always wins in a bandwidth contest.

🌐 Remote Access — Tailscale (How SJ Connects)

SJ uses Tailscale — the ASUS VPN server is not needed for remote access

SJ's remote access to SJAI runs via Tailscale, a zero-config mesh VPN built on WireGuard. It connects her phone, work laptop, and SJAI into one private network from anywhere. No DDNS, no port forwarding, no ASUS VPN config required. The double-NAT topology is completely irrelevant — Tailscale handles NAT traversal automatically.

Setup on SJAI (the supercomputer)

  1. Install Tailscale on SJAI: curl -fsSL https://tailscale.com/install.sh | sh
  2. Authenticate: tailscale up — SJAI gets a stable 100.x.x.x address that never changes
  3. SJ's phone and other devices already on Tailscale connect directly to SJAI's Tailscale IP from anywhere
  4. No changes needed on the ASUS router — default firewall allows all outbound traffic
Subnet Router — expose all 192.168.2.x devices via Tailscale

To make every device on SJ's subnet accessible remotely (not just SJAI), run SJAI as a subnet router:

tailscale up --advertise-routes=192.168.2.0/24

Approve the route in the Tailscale admin console. The NAS, any future lab machines, all devices on 192.168.2.x become reachable from SJ's Tailscale network.

💾 USB Ports — Full Feature Breakdown

The RT-AC86U has USB 3.0 (rear top) and USB 2.0 (rear bottom). These unlock 5 distinct features. Here's every one of them:

🗄️

Network Attached Storage (NAS) — Best Use

Plug a USB 3.0 SSD or HDD. Go to USB Application → Servers Center → Network Place (Samba). Enable sharing. Creates a shared drive accessible by any device on SJ's subnet (192.168.2.x). Format the drive as NTFS for Windows compatibility or ext4 for Linux performance. Gigabit LAN to the NAS means real-world 80–100 MB/s read speeds for a fast USB 3.0 SSD. This is a proper NAS for SJ's model files, datasets, and outputs — no extra hardware needed.

☁️

AiCloud 2.0 — Remote File Access

Go to AiCloud 2.0 → Cloud Disk → Enable. Once on, SJ can access the USB drive's files from ANY device globally using the ASUS AiCloud app or ASUS DDNS URL. Think of it as a private Dropbox running on the router. Useful for quickly pulling a model checkpoint from anywhere. Speed is limited by Contact upload speed (your WAN upload) — typically 50–100 Mbps on NZ fibre.

⬇️

Download Master — Background BitTorrent / Direct Downloads

Install via USB Application → Download Master. Runs on the router CPU — SJAI can be off and downloads still happen. Add magnet links or direct URLs. Files go straight to the USB drive. Useful for downloading large AI datasets (Hugging Face, Kaggle, Common Crawl) overnight without keeping SJAI powered. Note: Router CPU is not fast — expect 10–30 MB/s. Fine for overnight jobs.

🖨️

Network Printer Server

Plug any USB printer into USB 2.0 port. Go to USB Application → Network Printer Server → Enable. Install ASUS EZ Printer Sharing utility on Windows or use standard LPR/LPD protocol. All devices on SJ's subnet can print wirelessly. For setup: Add printer → TCP/IP port → enter ASUS IP → protocol LPR → queue name LPRServer → install printer driver.

📱

3G/4G USB Modem — Failover Internet (Dual WAN)

The RT-AC86U supports USB cellular modems as a secondary WAN source. Go to WAN → Dual WAN. If the TP-Link backhaul drops (ethernet cut, ISP outage), the router automatically switches to the cellular modem. Keeping SJAI always online during an outage. Requires a compatible USB 4G dongle (Huawei E8372 is widely supported). This is the "always-on AI" insurance policy.

USB Performance Tips

Always use USB 3.0 port for storage (not USB 2.0 — that caps at ~40 MB/s). Format drives as NTFS or ext4 (not FAT32 — 4GB file size limit will break large model files). Set the USB drive as a Static IP device in DHCP so its address never changes. This ensures Samba/AiCloud paths stay consistent.

🚨 Critical: Final Connectivity Fixes

Based on the latest audit, these three manual overrides are required to restore internet to the lab and secure the network.

1. The "Static Bridgehead" (Restore Internet)

The TP-Link is currently failing to hand an IP to the ASUS via DHCP. We will force the connection manually.

  1. Go to Advanced Settings → WAN (bottom left menu).
  2. Set WAN Connection Type to Static IP.
  3. Enter these exact values in the fields that appear:
    • IP Address: 192.168.1.50
    • Subnet Mask: 255.255.255.0
    • Default Gateway: 192.168.1.1
  4. Under WAN DNS Setting, set Connect to DNS Server automatically to No.
  5. Enter DNS Server 1: 1.1.1.1 and DNS Server 2: 1.0.0.1.
  6. Scroll to the very bottom and click Apply.
  7. Note: After clicking Apply, the "Internet Status" on the Network Map should change to "Connected" within 30 seconds.
2. Firewall Activation (Security)

The audit revealed your hardware firewall is currently DISABLED. We must fix this to protect SJAI.

  1. Go to Advanced Settings → Firewall.
  2. On the General tab, look for Enable Firewall.
  3. Click Yes (ensure the dot moves to Yes).
  4. Set Enable DoS protection to Yes.
  5. Click Apply at the bottom.
3. LAN DHCP Alignment (SJAI Local IP)

Ensure the ASUS is "handing out" addresses to your connected devices correctly.

  1. Go to Advanced Settings → LAN.
  2. Select the DHCP Server tab at the top.
  3. Set Enable the DHCP Server to Yes.
  4. Check the IP Pool Starting Address — it should be 192.168.2.2.
  5. Click Apply.
